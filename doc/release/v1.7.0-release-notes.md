# Zotero XMnote Plugin v1.7.0

## 中文

### 主要更新

- 支持 Zotero 7、8、9、10。
- Zotero 8–10 使用原生 `Zotero.MenuManager`，Zotero 7 自动使用兼容菜单实现。
- 同步当前纸间书摘 API：补充 `coverBase64`、书评和完整阅读状态等数据契约。
- 支持直接粘贴完整导入 URL，也兼容域名、IPv4、IPv6、HTTPS 和原有 IP + 端口配置。
- 连接测试改为只读 `OPTIONS` 探测，不再创建 `Connection Test` 测试书籍。
- 发送前增加页码、位置、阅读状态、封面和阅读时长校验，避免无效数据进入待导入列表。
- 正确区分“API 已接收”和“App 已正式导入”：`code=200` 表示进入待确认列表，仍需在纸间书摘中手动确认。
- 部分条目失败时不再误报整体成功，并展示已接收、失败数量和错误信息。
- 阅读时长估算改为每日模糊时长，不再冒充精确阅读会话。
- 不再自动猜测阅读状态，也不会在没有可靠标注时伪造当前页为第 1 页。
- 重写中文和英文 README，明确安装、配置、导出流程和产品边界。

### 兼容性验证

- Zotero 7.0.32：通过
- Zotero 8.0.5：通过
- Zotero 9.0.6：通过
- Zotero 10：通过 GitHub CI 验证
- 自动测试：21 项通过
- 真实纸间书摘 `/send` 请求：返回 `code=200`

### 使用提示

1. 在纸间书摘的“API 导入”页面启动服务。
2. 将 App 显示的完整地址粘贴到插件的 **Server Address / URL**。
3. 在 Zotero 中发送条目。
4. 在纸间书摘的待确认列表检查并手动确认导入。

本插件仍是单向导出工具，不提供双向同步，也不会读取或删除纸间书摘中的已有书籍。

## English

### Highlights

- Added support for Zotero 7, 8, 9, and 10.
- Zotero 8–10 use the native `Zotero.MenuManager`; Zotero 7 automatically falls back to the compatibility menu implementation.
- Updated the XMnote API contract with `coverBase64`, reviews, and the complete reading-status range.
- Added support for complete import URLs, hostnames, IPv4, IPv6, HTTPS, and the legacy IP + port configuration.
- Replaced the side-effecting `Connection Test` import with a read-only `OPTIONS` reachability probe.
- Added validation for pages, locations, reading status, covers, and reading durations before sending.
- Correctly distinguishes “accepted by the API” from “imported into the app.” `code=200` means the item is pending confirmation in XMnote.
- Partial failures now report accepted and failed counts instead of being presented as a complete success.
- Reading-duration estimates are exported as fuzzy daily durations, never as precise reading sessions.
- The plugin no longer invents a reading status or defaults the current page to page 1 without reliable annotation data.
- Rewrote both Chinese and English READMEs with accurate installation, configuration, workflow, and product boundaries.

### Verification

- Zotero 7.0.32: passed
- Zotero 8.0.5: passed
- Zotero 9.0.6: passed
- Zotero 10: passed in GitHub CI
- Automated tests: 21 passed
- Live XMnote `/send` request: returned `code=200`

### Usage

1. Start the service from XMnote's **API Import** page.
2. Paste the complete address shown by the app into **Server Address / URL**.
3. Send items from Zotero.
4. Review and confirm the pending import in XMnote.

This plugin remains a one-way export tool. It does not provide bidirectional synchronization and cannot read or delete books already stored in XMnote.
